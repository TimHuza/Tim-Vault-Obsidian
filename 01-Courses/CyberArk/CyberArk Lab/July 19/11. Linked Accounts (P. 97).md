# What is a Linked Account in CyberArk?

A **Linked Account** is an account that is **attached (linked) to another account** because CyberArk needs to use one account to perform actions on another account.

The main idea:

> **One account gives permission or helps manage another account.**

Think of it like a **key and a lock**.

- The **main account** = the account you want to manage.
    
- The **linked account** = the helper account that allows CyberArk to perform a specific action.
    

---

# Simple Real-Life Example

Imagine you have a bank vault.

There are two people:

### Person A: Bank Manager

- Has the authority to open the vault.
    
- Can change locks.
    
- Has special permissions.
    

### Person B: Regular Employee

- Works inside the bank.
    
- Needs help from the manager for certain actions.
    

The employee account is **linked** to the manager account because the manager enables special operations.

CyberArk works similarly.

---

# Why Do We Need Linked Accounts?

CyberArk manages privileged accounts, but sometimes **one account does not have enough permissions** to perform an operation.

For example:

A Windows Administrator account:

```
Username:
Administrator

Password:
********
```

can log into a server.

But CyberArk wants to change the password of another account:

```
Username:
SQL_Service
```

The Administrator account might be required to change the SQL_Service password.

So CyberArk creates a relationship:

```
Administrator Account
        |
        |
        ↓
Linked Account
        |
        |
        ↓
SQL_Service Account
```

Meaning:

> "Use Administrator to manage SQL_Service."

# How Linked Accounts Work in CyberArk

Let's walk through a common scenario.

## Step 1 — You Store an Account in CyberArk

Example:

Safe:

```
Windows_Admin_Safe
```

Contains:

```
Account:
Administrator

Platform:
Windows Domain Account

Address:
SERVER01.company.com
```

CyberArk manages this password.

---

## Step 2 — CyberArk Needs to Change Another Account

You onboard:

```
Account:
SQL_Service

Address:
SERVER01.company.com
```

CyberArk tries to change the password.

But:

```
SQL_Service
```

does not have permission to change its own password.

---

## Step 3 — Create a Linked Account

You tell CyberArk:

"To manage SQL_Service, use Administrator."

So you create:

```
SQL_Service Account
        |
        |
        ↓
Linked Account:
Administrator
```

Now CyberArk knows:

```
I need to change SQL_Service password.

Who can do it?

→ Use Administrator account.
```

---

## Step 4 — CPM Uses the Linked Account

The CyberArk Password Manager (CPM) performs the password operation.

The flow:

```
                 CyberArk Vault
                      |
                      |
                      ↓
              Password Manager (CPM)
                      |
                      |
                      ↓
          Needs to change SQL_Service password
                      |
                      |
                      ↓
          Uses Linked Account credentials
                      |
                      |
                      ↓
          Administrator account
                      |
                      |
                      ↓
          Changes SQL_Service password
```

# Linked Account vs Normal Account

## Normal Account

A normal CyberArk account:

```
Account
 |
 ↓
Administrator
```

CyberArk stores and manages it.

---

## Linked Account

A linked account:

```
Account A
   |
   |
   ↓
Account B
```

Account A depends on Account B.

Example:

```
SQL_Service
     |
     |
     ↓
Administrator
```

Meaning:

"Use Administrator to manage SQL_Service."

---

# Where Are Linked Accounts Used?

Common examples:

---

## 1. Password Reconciliation

Very common.

Example:

CyberArk tries to change:

```
Database_Service
```

but the password is unknown.

A reconciliation account can fix it.

Example:

```
Database_Service
        |
        |
        ↓
Domain_Admin
```

Domain_Admin resets the password.

---

## 2. Windows Services

Example:

A Windows service runs with:

```
svc_backup
```

CyberArk needs to change:

```
svc_backup password
```

but changing it requires administrator rights.

So:

```
svc_backup
      |
      |
      ↓
Administrator
```

---

## 3. Database Accounts

Example:

Oracle database user:

```
oracle_admin
```

requires another privileged account to modify it.

---

# Linked Account vs Reconcile Account

Beginners often confuse these.

## Linked Account

Purpose:

> "Use this account to perform an action."

Example:

```
Service Account
       |
       ↓
Administrator
```

---

## Reconcile Account

Purpose:

> "Fix or reset a password when CyberArk cannot manage it."

Example:

CyberArk says:

```
Password is wrong.
I cannot log in.
```

Then:

```
Reconcile Account
        |
        ↓
Resets password
```

# The Main Idea of Linked Accounts

Remember this sentence:

> **A Linked Account is a relationship between two accounts where one account is used by CyberArk to manage another account.**

Or even simpler:

> **"CyberArk uses Account A to control Account B."**

---

For a CyberArk beginner, the most important thing to remember:

|Term|Meaning|
|---|---|
|Account|Password CyberArk stores|
|Linked Account|Helper account connected to another account|
|CPM|Component that performs password changes|
|Reconcile Account|Emergency account that fixes password problems|
|Platform|Rules telling CyberArk how to manage the account|

In a real enterprise environment (like a bank), Linked Accounts are extremely common because many service accounts **cannot manage themselves**, so CyberArk needs a more powerful account to perform the operation.
