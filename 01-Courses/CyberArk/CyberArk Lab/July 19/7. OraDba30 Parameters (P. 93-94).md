# First: What is happening here?

Imagine a company has a database server:

```
Oracle Database Server
        |
        |
   Admin Account
   (root / sysdba)
```

The administrator should **not directly connect** to the database server.

Instead, CyberArk forces the connection through:

```
User
 |
 |
PVWA
 |
 |
PSM
 |
 |
Oracle Database Server
```

The PSM server acts like a **security camera + jump server**.

It does two important things:

1. Allows the user to connect safely.
    
2. Records everything the user does.
    

Example:

A DBA connects:

```
John
 |
 |
CyberArk PVWA
 |
 |
PSM Server
 |
 |
Oracle Server
```

PSM records:

- Who connected
    
- When they connected
    
- What commands they typed
    
- What screens they saw
    
- How long they stayed connected
    

That recording is stored in a special Vault Safe.

This is where `ID` and `SessionRecordSafe` come in.

---

# 1. ID Parameter

From your screenshot:

```
ID = PSMServer_COMP02
```

## What is ID?

`ID` is the **unique name of the PSM server** that will handle the session.

Think of it like a computer name.

Example:

A company has:

```
PSM Servers:

PSMServer_COMP01
PSMServer_COMP02
PSMServer_COMP03
```

CyberArk needs to know:

> "Which PSM server should I use when someone connects?"

The `ID` tells CyberArk:

> "Use this specific PSM server."

---

## Simple example

You have 3 security guards:

```
Guard A
Guard B
Guard C
```

Your company says:

"All visitors must go through Guard B."

The name of Guard B is:

```
PSMServer_COMP02
```

So CyberArk stores:

```
ID = PSMServer_COMP02
```

Meaning:

"Send sessions to this PSM server."

---

## How does it work?

User clicks:

```
Connect
```

in PVWA.

CyberArk checks:

```
Platform:
    OraDba30

PSM Settings:
    ID = PSMServer_COMP02
```

Then:

```
PVWA
 |
 |
Find PSMServer_COMP02
 |
 |
Start session
 |
 |
Oracle Database
```

---

## Why do we need ID?

Because large companies have many PSM servers.

Example:

A bank:

```
USA Data Center

PSM01
PSM02


Europe Data Center

PSM03
PSM04
```

CyberArk needs to know:

"Which PSM should handle this connection?"

The ID provides that information.

# 2. SessionRecordSafe Parameter

Your screenshot:

```
SessionRecordSafe = PSMR_OraDba30
```

This one is very important.

---

## What is SessionRecordSafe?

`SessionRecordSafe` is the **CyberArk Vault Safe where PSM recordings are stored**.

Think about it like a security camera storage room.

Example:

A camera records:

```
John logged in
10:00 AM

Opened SQLPlus

Typed:
DROP TABLE USERS;

Disconnected
```

Where do we store this video?

Inside:

```
PSMR_OraDba30 Safe
```

---

## What is a Safe?

In CyberArk:

A **Safe** is a secure container inside the Vault.

It stores:

- Passwords
    
- Recordings
    
- Files
    
- Logs
    

Example:

```
CyberArk Vault

|
|-- Safe: Windows_Admin
|       |
|       |-- Administrator password
|
|
|-- Safe: Oracle_Admin
|       |
|       |-- SYS password
|
|
|-- Safe: PSMR_OraDba30
        |
        |-- Session recordings
```

---

# How SessionRecordSafe works step-by-step

Let's say:

A DBA connects to Oracle.

---

### Step 1

User logs into PVWA:

```
John
```

requests:

```
Oracle Admin Account
```

---

### Step 2

PVWA sends connection to PSM:

```
PVWA
 |
 |
PSMServer_COMP02
```

because:

```
ID = PSMServer_COMP02
```

---

### Step 3

PSM opens the Oracle session:

```
PSM
 |
 |
Oracle Database
```

---

### Step 4

PSM starts recording:

```
Recording starts:

John
10:01

Commands:
SELECT *
UPDATE USER_TABLE

Screen:
(all captured)
```

---

### Step 5

When session ends:

PSM saves the recording:

```
Session Recording
        |
        |
        v

Vault Safe:
PSMR_OraDba30
```

because:

```
SessionRecordSafe = PSMR_OraDba30
```

---

# Why do we need SessionRecordSafe?

Main reasons:

## 1. Auditing

A company can answer:

> "Who accessed the database yesterday?"

CyberArk shows:

```
User:
John Smith

Server:
Oracle Production

Time:
14:35

Recording:
Available
```

---

## 2. Security investigations

Imagine:

Someone deletes important data.

Security team asks:

"Who did this?"

They can watch the recording:

```
PSMR_OraDba30 Safe

Recording:
John - Oracle Session
```

---

## 3. Compliance

Banks and governments often require:

- Session recording
    
- User tracking
    
- Privileged activity monitoring
    

Examples:

- PCI-DSS
    
- SOX
    
- ISO 27001

# Difference between ID and SessionRecordSafe

|Parameter|Meaning|Example|
|---|---|---|
|**ID**|Which PSM server handles the connection|`PSMServer_COMP02`|
|**SessionRecordSafe**|Where the recording is stored|`PSMR_OraDba30`|
