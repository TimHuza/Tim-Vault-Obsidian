#cyberark-labs 

# Short Answer

There are actually **two different things**:

1. 📝 **Logs** (text information)
    
2. 🎥 **Session Recordings** (video-like recordings)
    

They are stored in **different places**.

---

# 1. Where are the Logs stored?

Logs are simply text.

For example:

```
User: Bob
Account: Administrator
Server: Server01
Login Time: 10:00 AM
Logout Time: 10:25 AM
Password Retrieved: Yes
Password Changed: Success
```

CyberArk stores these audit logs in its components, primarily in the **CyberArk Vault** and in the databases used by CyberArk components (depending on the component and configuration).

Think of it like a notebook.

```
CyberArk Vault
│
├── Passwords
├── Audit Logs
└── Configuration
```

Whenever someone does something in CyberArk:

- Opens PVWA
    
- Retrieves a password
    
- Starts a session
    
- Changes a password
    
- Creates a Safe
    

CyberArk writes it into its audit log.

---

# 2. Where are Session Recordings stored?

This is different.

Imagine Bob opens an RDP session through **PSM**.

```
Bob
   │
   ▼
PVWA
   │
   ▼
PSM
   │
   ▼
Windows Server
```

While Bob is working...

PSM records everything.

It is almost like someone is recording your screen.

The recording contains things like:

- Mouse movements
    
- Keyboard typing
    
- Windows opened
    
- Commands executed
    

---

After the session ends...

PSM uploads the recording into the **CyberArk Vault**.

```
            +-------------------+
            |  CyberArk Vault   |
            |-------------------|
            | Passwords         |
            | Audit Logs        |
            | Session Recording |
            +-------------------+
```

The Vault becomes the secure storage location.

---

# Why store recordings in the Vault?

Imagine if the recordings stayed only on the PSM server.

If someone hacked the PSM server, they could:

- Delete recordings ❌
    
- Modify recordings ❌
    
- Hide evidence ❌
    

Instead, PSM sends them to the Vault.

The Vault is designed to be highly secure.

Once stored there, recordings are much harder to tamper with.

---

# Real Example

Bob connects to a production server.

```
9:00
Bob starts session

↓

PSM begins recording

↓

Bob installs software

↓

Bob restarts server

↓

Bob logs out

↓

PSM finishes recording

↓

Recording stored securely in Vault
```

Months later...

An Auditor can search for:

```
User:
Bob

Date:
July 20

Server:
Windows-Prod-01
```

Then click **Play Recording**.

It looks similar to watching a recorded remote desktop session.

---

# How does an Auditor watch it?

The Auditor logs into **PVWA**.

```
Auditor
    │
    ▼
PVWA
    │
    ▼
Recorded Sessions
    │
    ▼
Search
    │
    ▼
Play Recording
```

The recording streams securely from the Vault through CyberArk. The Auditor doesn't normally browse the Vault directly.

---

# What if there are thousands of sessions?

CyberArk indexes them.

An Auditor can search by:

- Username
    
- Safe
    
- Account
    
- Server
    
- Date
    
- Time
    
- Platform
    
- Duration
    

For example:

```
Find all sessions

User = Bob

Date = Yesterday
```

CyberArk immediately lists the matching recordings.

## One important note

In most CyberArk deployments, **session recordings are securely stored in the CyberArk Digital Vault after PSM uploads them**. Some organizations may configure additional archival or backup storage for long-term retention, but the Vault is the primary secure repository that CyberArk uses to protect those recordings.
