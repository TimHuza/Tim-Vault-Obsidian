# First, what is an Auditor?

Imagine your school has a **treasure room** full of exam answers.

Only teachers are allowed inside.

But the principal wants to make sure **teachers are using the room properly**.

Instead of standing there all day, the principal hires someone whose only job is to **watch what happened**.

That person is the **Auditor**.

The Auditor **does NOT**:

- Change passwords ❌
    
- Use privileged accounts ❌
    
- Fix servers ❌
    

The Auditor's job is simply to **look at records and reports**.

---

# What does an Auditor do in CyberArk?

CyberArk stores passwords for very important accounts like:

- Windows Administrator
    
- Linux root
    
- Database Administrator
    
- Domain Administrator
    

People use these accounts every day.

Someone needs to answer questions like:

- Who logged in?
    
- When?
    
- Which account did they use?
    
- Did they follow company rules?
    
- Did anyone do something suspicious?
    

That's exactly what the Auditor checks.

---

# Think of a Security Camera

Imagine a bank.

There are security cameras everywhere.

The cameras don't stop robberies.

They simply record everything.

Later, if something happens, security can watch the recordings.

An Auditor is very similar.

CyberArk records almost everything.

The Auditor looks through those records.

# Example

Suppose Bob is a system administrator.

He wants to log in as **Administrator**.

```
Bob
   │
   ▼
CyberArk
   │
Gets Administrator password
   │
   ▼
Windows Server
```

Bob works on the server for 20 minutes.

CyberArk records things like:

- Bob logged in
    
- Date
    
- Time
    
- Which account
    
- Which server
    
- How long the session lasted
    

The Auditor can later see all of that.

# What if Bob does something bad?

Imagine Bob accidentally deletes an important folder.

The company asks:

> "Who deleted it?"

The Auditor opens CyberArk logs.

They see:

```
User:
Bob

Account:
Administrator

Server:
Server01

Login:
10:05 AM

Logout:
10:32 AM
```

If the session was recorded through **PSM (Privileged Session Manager)**, the Auditor can even **play back the entire session like a video** to see exactly what Bob clicked.

# What information can an Auditor see?

An Auditor can view things like:

- Login history
    
- Password changes
    
- Password verification results
    
- Password reconciliation events
    
- Session recordings
    
- Failed login attempts
    
- Policy violations
    
- Reports
    
- Audit logs
    

They usually **cannot** retrieve passwords or perform administrative changes—they focus on reviewing activity.

# Why is the Auditor important?

Without an Auditor:

❌ Nobody knows who used privileged accounts.

❌ Nobody knows if someone broke company policy.

❌ Nobody can investigate security incidents.

---

With an Auditor:

✅ Every privileged action is traceable.

✅ Companies can investigate incidents.

✅ Security policies can be verified.

✅ Organizations can meet compliance requirements (such as financial or healthcare regulations).

# Easy way to remember

Think of CyberArk like a bank:

- 🏦 **Vault** = Stores the passwords.
    
- 👨‍💻 **Administrator** = Manages the CyberArk system.
    
- 🔄 **CPM** = Automatically changes passwords.
    
- 🎥 **PSM** = Records what users do during privileged sessions.
    
- 🕵️ **Auditor** = Reviews the recordings and logs to make sure everything was done correctly.
    

**One sentence to remember:**

> **An Auditor is the detective of CyberArk. They don't use privileged accounts—they investigate who used them, when they used them, and what they did.**
