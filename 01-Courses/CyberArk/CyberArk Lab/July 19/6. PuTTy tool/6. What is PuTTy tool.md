PuTTY is a **free tool that allows you to connect to another computer/server remotely using a network connection**.

Think of PuTTY as a **remote keyboard and screen** for another machine.

Imagine you have:

```
Your computer (Windows)
        |
        |  Network connection (SSH)
        |
        v
Linux Server / CyberArk Server
```

You sit on your Windows computer, open PuTTY, connect to the server, and you get a **command-line window** where you can type commands as if you were sitting in front of that server.

---

### Simple example

You have a Linux server in your CyberArk lab:

```
Server:
IP: 192.168.1.50
Username: admin
Password: ********
```

You open PuTTY:

```
PuTTY
 |
 |-- Host Name: 192.168.1.50
 |-- Port: 22
 |-- Connection type: SSH
 |
 Connect
```

PuTTY opens:

```
login as: admin
admin@192.168.1.50's password:

[user@server ~]$
```

Now you are controlling the Linux server remotely.

You can run:

```bash
ls
```

to see files:

```
Documents
Logs
Scripts
```

or:

```bash
systemctl status sshd
```

to check a service.

## [[6.2. Why Do We Need PuTTy in CyberArk|Why]] is PuTTY used in CyberArk?

In CyberArk environments, administrators often need to connect to:

- Linux servers
    
- Unix servers
    
- Network devices
    
- Security appliances
    

Many of these systems use **SSH**.

PuTTY is one of the tools used to create that SSH connection.

Example:

```
Administrator Laptop
        |
        |
      PuTTY
        |
        | SSH
        |
        v
Linux Server
```

---

## PuTTY is NOT:

❌ Not a server  
❌ Not a password manager  
❌ Not CyberArk itself  
❌ Not storing passwords

It is just a **client application** used to connect to other machines.

---

## [[6.1. How PuTTy Works With Platform|How]] this relates to CyberArk PSM

In a normal environment:

```
User
 |
 |
PuTTY
 |
 |
Linux Server
```

The user directly connects.

But with CyberArk:

```
User
 |
 |
PVWA
 |
 |
PSM
 |
 |
Linux Server
```

CyberArk puts **PSM (Privileged Session Manager)** in the middle.

The user does not connect directly with PuTTY.

Instead:

1. User logs into CyberArk PVWA
    
2. User requests access to a Linux account
    
3. CyberArk PSM opens the session
    
4. PSM uses SSH (similar to PuTTY) to connect to the Linux server
    
5. CyberArk records the session
    

So:

**PuTTY = a tool humans use to SSH into servers**  
**CyberArk PSM = CyberArk's controlled and recorded way to do the same thing**

A simple comparison:

|Tool|Purpose|
|---|---|
|PuTTY|Human manually connects to servers|
|PSM|CyberArk securely controls and records connections|
|SSH|The communication protocol used to connect|

For your CyberArk lab, remember:

> **PuTTY is basically a remote terminal program used to connect to Linux/Unix machines using SSH.**
