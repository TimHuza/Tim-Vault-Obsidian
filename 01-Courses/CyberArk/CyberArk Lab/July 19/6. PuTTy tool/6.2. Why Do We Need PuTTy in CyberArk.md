#cyberark-labs 


> **We need PuTTY in CyberArk because many privileged systems (especially Linux/Unix servers) require an SSH client to connect. PuTTY is one possible SSH client that CyberArk PSM can use to open and control those sessions.**
## 1. The problem CyberArk solves

Before CyberArk, an administrator might do this:

```
Administrator
      |
      |
    PuTTY
      |
      | SSH
      |
      v
 Linux Server

Username: root
Password: MySecretPassword123
```

Problems:

❌ The administrator knows the password  
❌ Passwords can be shared  
❌ No recording of what the admin did  
❌ Hard to audit who accessed the server

---

## 2. CyberArk changes the process

With CyberArk:

```
User
 |
 |
PVWA
 |
 |
PSM
 |
 |
Linux Server
```

The user does not need to know the password.

CyberArk says:

> "I will take care of the password and create the connection for you."

---

## 3. Where PuTTY fits

Linux servers commonly use **SSH**.

SSH needs a client program.

Examples of SSH clients:

- PuTTY
    
- OpenSSH
    
- SecureCRT
    
- MobaXterm
    

PuTTY is just one of them.

CyberArk PSM needs something that can create the SSH connection.

Example:

```
PSM Server
     |
     |
     | launches PuTTY
     |
     |
     | SSH connection
     |
     v
Linux Server
```

PuTTY creates the technical connection.

CyberArk controls the security around it.

---

## 4. What happens when you click "Connect" in PVWA?

Example:

You have this account:

```
Account:
root@Linux01

Platform:
Unix via SSH
```

You click:

```
Connect
```

Behind the scenes:

### Step 1

CyberArk checks the Platform:

```
Platform:
Unix via SSH

Connection:
SSH

Port:
22
```

It learns:

> "This is a Linux SSH connection."

---

### Step 2

PSM starts a session.

```
PSM
 |
 |
 SSH connection
 |
 |
Linux Server
```

---

### Step 3

PSM may open PuTTY:

```
PuTTY window

root@Linux01:
[root@Linux01 ~]#
```

The user sees the terminal.

But:

- CyberArk entered the password
    
- CyberArk controls the session
    
- CyberArk records the session
    

---

## 5. Why not just connect directly?

Without CyberArk:

```
User
 |
 |
PuTTY
 |
 |
Linux Server
```

The user owns the connection.

With CyberArk:

```
User
 |
 |
PVWA
 |
 |
PSM
 |
 |
PuTTY
 |
 |
Linux Server
```

CyberArk owns and monitors the connection.

---

## In one sentence:

**PuTTY is needed in CyberArk because it is an SSH connection tool that allows PSM to open sessions to Linux/Unix systems, while CyberArk controls the access, hides the password, and records the activity.**

For your lab, remember this chain:

```
Platform
   ↓
Tells CyberArk: "Use SSH"
   ↓
PSM
   ↓
Uses PuTTY/SSH client
   ↓
Linux Server
```

PuTTY is not the security part — **PSM is the security part. PuTTY is only the tool that helps create the connection.**
