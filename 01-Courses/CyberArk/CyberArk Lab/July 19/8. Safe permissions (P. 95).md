You are looking at a **CyberArk Platform parameter** inside:

```
Platform Management
    |
    └── Automatic Password Management
            |
            └── Additional Policy Settings
                    |
                    └── ConnectionCommand
```

This is part of the **CPM (Central Policy Manager)** configuration.

Remember the big picture:

```
User
 |
PVWA
 |
CPM  --->  Connects to target system
              |
              |
              +---- Changes password
              +---- Verifies password
              +---- Reconciles password
```

The **CPM needs instructions on HOW to connect to the target system**.  
`ConnectionCommand` is one of those instructions.

---

# 1. What is `ConnectionCommand`?

## Simple explanation

`ConnectionCommand` tells CyberArk CPM:

> "When you need to connect to this target system, use this command."

Think of it like a **recipe**.

Imagine you want to call your friend:

```
Phone number: 555-1234
Action: Call
```

The phone number tells you **how to reach the person**.

In CyberArk:

```
ConnectionCommand = How CPM reaches the database/server
```

---

## Why does CPM need it?

CPM does not magically know:

- Is this Oracle?
    
- Is this SQL Server?
    
- Is this Windows?
    
- Is this Linux?
    
- What tool should I use?
    
- What driver should I load?
    

The Platform tells CPM:

> "For this type of account, use this connection method."

Example:

For an Oracle database account:

```
Username:
SYSTEM

Password:
********

Target:
Oracle Database

ConnectionCommand:
sqlplus.exe
```

CPM uses that information to connect.

---

# How does `ConnectionCommand` work?

Let's follow the password change process.

Example:

You have an Oracle account:

```
Account:
ORA_DBA

Username:
system

Password:
OldPassword123
```

CyberArk wants to change the password.

---

## Step 1 - CPM receives the task

CPM says:

> "I need to change the password for ORA_DBA."

---

## Step 2 - CPM reads the Platform settings

It checks:

```
Oracle Platform
     |
     |
     +-- Automatic Password Management
              |
              +-- ConnectionCommand
```

It finds:

```
ConnectionCommand =
Driver={Oracle in instantclient_21_20};
...
```

---

## Step 3 - CPM builds the connection

It combines:

```
ConnectionCommand
+
Username
+
Password
+
Database address
+
Database name
```

and creates a database connection.

Example:

```
Connect to:

Oracle DB
192.168.1.50
PORT 1521

using:

username = system
password = OldPassword123
```

---

## Step 4 - CPM changes password

It runs something like:

```sql
ALTER USER SYSTEM IDENTIFIED BY NewPassword456;
```

Oracle changes the password.

---

## Step 5 - CyberArk verifies

CPM reconnects:

```
Username:
system

Password:
NewPassword456
```

If successful:

```
Password change successful
```

# 2. What is:

```
Driver={Oracle in instantclient_21_20}
```

?

This is the **database driver information**.

---

## Simple explanation

A driver is a translator.

Imagine:

You speak English.

Oracle database speaks "Oracle protocol".

The driver translates:

```
CPM
 |
 | English
 |
Oracle Driver
 |
 | Oracle language
 |
Oracle Database
```

Without the driver:

```
CPM --------------X-------------- Oracle
       cannot communicate
```

---

# What is Oracle Instant Client?

Oracle Instant Client is software from Oracle.

It contains tools and libraries that allow applications to connect to Oracle databases.

Example:

```
CPM Server
 |
 |
 Oracle Instant Client
 |
 |
 Oracle Database
```

---

Your parameter:

```
Driver={Oracle in instantclient_21_20}
```

means:

> "Use the Oracle ODBC driver from Oracle Instant Client version 21.20."

---

# Breaking it down

## Driver

```
Driver={}
```

means:

"Load this database communication driver."

---

## Oracle

```
Oracle
```

means:

"This driver is for Oracle databases."

---

## instantclient_21_20

```
instantclient_21_20
```

means:

"The driver comes from Oracle Instant Client version 21.20."

---

So:

```
Driver={Oracle in instantclient_21_20}
```

means:

> "When CPM connects to Oracle, use the Oracle Instant Client 21.20 driver."

---

# Why do we need the driver?

Because CPM is a Windows/Linux application.

It does not natively understand every database.

For example:

CPM wants to say:

```
Change password for user SCOTT
```

But Oracle expects a special protocol.

The driver converts:

```
CyberArk command

       ↓

Oracle driver

       ↓

Oracle database command
```

# What happens if the driver is missing?

Example:

CyberArk Platform says:

```
Driver={Oracle in instantclient_21_20}
```

but CPM server does not have Oracle Instant Client installed.

Then:

```
CPM
 |
 |
 Looking for Oracle Driver
 |
 |
 ERROR:
 Driver not found
```

Password management fails.

You might see errors like:

```
Cannot connect to database
ODBC driver not found
```

---

# ConnectionCommand vs Driver (difference)

|Parameter|Meaning|
|---|---|
|ConnectionCommand|The instructions for creating the connection|
|Driver|The software component that actually communicates with Oracle|

Think about a car:

```
ConnectionCommand
=
GPS route

Driver
=
Engine
```

The route tells you where to go.

The engine makes the car move.

You need both.
