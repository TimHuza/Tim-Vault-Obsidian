#cyberark-labs 

![[CyberArk Enterprise Architecture in Bank Environment.png]]

Think of a **bank** that has thousands of important passwords:

- Windows Administrator passwords
    
- Linux root passwords
    
- Database passwords
    
- Network device passwords
    

The bank **cannot write these passwords on paper** or let everyone know them.

So they build a **super secure password city** called **CyberArk**.

Let's walk through your picture from **top to bottom**.

---

# Step 1. The People (Top of the picture)

At the very top you see

```
Bank Users / Admins
Auditors / Security Team
```

These are people.

Examples:

👨‍💻 Windows Administrator

👩‍💻 Linux Administrator

👨 Database Administrator

👩 Security Team

Nobody talks directly to servers.

Instead they all go here:

```
PVWA
```

Think of PVWA as the **front door** to CyberArk.

---

# Step 2. PVWA (Purple Box)

PVWA means

> Password Vault Web Access

It is simply a website.

Imagine opening Chrome.

Instead of typing

```
google.com
```

you type

```
https://cyberark.bank.com
```

You login.

Now you can:

- Search accounts
    
- Request passwords
    
- Start remote sessions
    
- View recordings
    
- Check policies
    

Everything starts here.

So in your picture

```
User
     ↓
    PVWA
```

means

> "Every administrator first logs into PVWA."

---

# Step 3. Primary Vault (Blue Safe)

This is the most important box.

Imagine a giant bank vault.

Inside are thousands of passwords.

```
Windows Admin
Linux Root
Oracle SYS
Cisco Admin
AWS Root
Azure Admin
```

Everything is stored here.

The Vault stores

✔ passwords

✔ SSH keys

✔ certificates

✔ policies

✔ audit logs

✔ account information

The Vault is like

> The bank's password safe.

Nobody can simply open it.

Even CyberArk components must ask permission.

---

# Step 4. Why does PVWA talk to the Vault?

When you login,

PVWA asks

> "Hey Vault...  
> Does Tim have permission to see this account?"

The Vault answers

```
Yes
```

or

```
No
```

If yes...

PVWA shows you the account.

If no...

Nothing appears.

The Vault makes all security decisions.

---

# Step 5. CPM (Green Box)

CPM means

Credential Provider Manager

Most beginners remember it as

> Password Robot

Imagine you have this password:

```
Administrator

Password123
```

That password is dangerous.

CyberArk says

> Change it.

Who changes it?

CPM.

CPM connects to the server.

```
Windows Server
```

Changes

```
Password123
```

into

```
T#8g!k92L@Px
```

Then updates the Vault.

Nobody had to do anything.

The robot did it.

---

### So the flow is

```
Vault
     ↓
CPM
     ↓
Windows Server
```

CPM changes passwords automatically.

---

# Step 6. PSM (Yellow Box)

This is one of the coolest parts.

PSM means

Privileged Session Manager

Think of PSM as a security guard.

Normally an admin would do

```
My PC
     ↓
Windows Server
```

CyberArk says

"No."

Instead

```
My PC
      ↓
     PSM
      ↓
 Windows Server
```

Everything goes through PSM.

---

Why?

Because PSM records everything.

Like YouTube.

Every click.

Every command.

Every window.

Everything.

If someone deletes a database...

Security can replay the recording.

---

# Step 7. PSMP (Purple Box)

PSMP is similar.

It is for SSH.

Instead of Remote Desktop...

Linux administrators use SSH.

Normally

```
Laptop
    ↓
Linux Server
```

CyberArk changes it into

```
Laptop
     ↓
PSMP
     ↓
Linux Server
```

PSMP

✔ checks permissions

✔ records activity

✔ hides passwords

# Step 9. DR Vault (Red Box)

Now imagine something terrible happens.

🔥 Fire

💥 Power outage

💧 Flood

💻 Hardware failure

The Primary Vault stops working.

Would the bank stop?

No.

That's why we have

```
DR Vault
```

DR means

Disaster Recovery

Think of it as

> A backup safe.

---

The Primary Vault constantly copies information.

```
Primary Vault
       ↓
Replication
       ↓
DR Vault
```

Every password.

Every account.

Every policy.

Every log.

Everything is copied.

---

# Step 10. Failover

Normally

```
Users

↓

Primary Vault
```

If the Primary Vault dies...

CyberArk switches to

```
Users

↓

DR Vault
```

This is called

Failover

Business continues.

Nobody notices much.

# An easy way to remember each component

|Component|Think of it as...|Job|
|---|---|---|
|**Primary Vault**|🏦 Bank safe|Stores all passwords and policies|
|**DR Vault**|🏦 Backup safe|Takes over if the main Vault fails|
|**PVWA**|🚪 Front door|Website where users log in|
|**CPM**|🤖 Password robot|Automatically changes and verifies passwords|
|**PSM**|🎥 Security guard|Opens RDP sessions and records everything|
|**PSMP**|🔐 SSH security guard|Opens SSH sessions securely and records them|

## One important concept

Many beginners think:

> **"PVWA stores passwords."**

It **does not**.

Think of it this way:

- **Vault** = The safe that **stores** passwords.
    
- **PVWA** = The website you use to **access** the safe.
    
- **CPM** = The robot that **changes** passwords.
    
- **PSM** = The guard that **controls and records** Windows/RDP sessions.
    
- **PSMP** = The guard that **controls and records** Linux/SSH sessions.
    

If you remember just those five roles, you'll already understand about **80% of a typical CyberArk enterprise architecture diagram**.
