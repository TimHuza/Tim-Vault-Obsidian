PA Workflows & Onboarding Rules and Account Discovery

### **1. Privileged Access Workflows**

These are the rules that govern how a person gets to use a "skeleton key" (privileged account).

- **Dual Approval:** This means one person cannot open a digital "vault" alone; they must request access, and a manager must **approve** the request first. The manager only sees the reason for the request and does not see the actual password.
- **Approval Requests:** These act as security checkpoints. Without them, anyone could use a powerful account at any time without a record of why.
- **Reason for Access:** In the lab, you configure a list of **predefined reasons** (like "Incident" or "Monthly patching") that users must select from before they can connect to a target machine.

### **2. Managing the Keys (Check-in and Release)**

CyberArk acts like a library for passwords where you "borrow" a key for a specific amount of time.

- **Check-in:** This is the button you click to **return a password early** so the next person can use it.
- **Exclusive Access:** When this is turned on, only one person can "borrow" the account at a time.
- **Automated Release by PSM:** If a user’s computer crashes or they just close their browser without logging out, the **PSM service** detects they are gone and automatically "checks the key back in" for them.
- **One-Time Passwords (OTP):** You can set a rule that as soon as a key is returned (released), the robot (CPM) **automatically changes the password** so it is never the same twice.

### **3. Working with Many Accounts (Bulk and Scope)**

When a company has thousands of accounts, you need tools to handle them all at once.

- **Bulk:** This means doing the same thing to many accounts together, like **changing 100 passwords** with one click instead of doing them one by one.
- **Scope:** This defines the **boundary** or "target area" where a rule applies. For example, a rule might apply to a whole "classroom" (Safe) or just one "student" (Account).

### **4. Finding and Protecting Accounts (Discovery and Onboarding)**

This is the process of finding "hidden" keys and bringing them into the digital fortress.

- **Organizational Units (OUs):** These are like **folders** inside a company's main phonebook (Active Directory) that organize users and computers.
- **Discovery:** This is a feature that **scans the network** to find powerful accounts that aren't protected yet.
- **Onboarding:** This is the act of **bringing an account under CyberArk's protection**. Once onboarded, the robot (CPM) starts managing its password automatically.
- **Dependencies:** In your lab, you see that onboarding a Windows account also discovers its **dependencies**, like scheduled tasks that need to be updated whenever the main password changes.
