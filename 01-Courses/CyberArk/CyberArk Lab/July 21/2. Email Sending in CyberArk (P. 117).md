#cyberark-labs 


CyberArk sends emails when something important happens, for example:

- A password is about to expire
    
- A password change failed
    
- Someone requested access to an account
    
- Someone approved/rejected a request
    
- A CyberArk component has a problem
    
- A privileged session started or ended
    

The main idea:

> **CyberArk does not send emails directly by itself. It asks an SMTP mail server to send emails for it.**

# 2. What is SMTP?

Before CyberArk, you need to understand email.

SMTP means:

**Simple Mail Transfer Protocol**

It is the language computers use to send emails.

Think:

```
CyberArk:
"Hey SMTP server, send this email"

SMTP Server:
"Okay, I will deliver it"
```

Example:

CyberArk creates:

```
From:
cyberark@company.com

To:
admin@company.com

Subject:
Password Change Failed

Message:
The password change for server01 failed.
```

Then SMTP delivers it.

# 3. Where do we configure email in CyberArk?

Usually in:

```
PVWA
 |
 └── Administration
       |
       └── Options
             |
             └── Notifications
```

You configure things like:

```
SMTP Server:
smtp.company.com

SMTP Port:
25

Sender Email:
cyberark@company.com
```

Example:

```
SMTP Server = mail.company.com
Port = 25
From Address = cyberark@company.com
```

CyberArk now knows:

"Whenever I need to send an email, I will talk to this SMTP server."

# 5. Example 1: Password Change Failed

Let's follow a real CyberArk event.

You have:

```
Windows Server
     |
     |
Domain Admin Account
     |
     |
CyberArk Vault
```

CyberArk CPM tries to change the password.

---

## Step 1

CPM says:

```
I need to rotate this password.
```

It connects:

```
CPM
 |
 |
 v
Windows Server
```

---

## Step 2

Password change fails.

Example:

```
Old password:
ABC123

New password:
XYZ789

Result:
FAILED
```

---

## Step 3

CPM reports the failure.

```
CPM
 |
 |
 v
Vault
```

Vault records:

```
Password Change Failed
Account: Administrator
Server: SERVER01
```

---

## Step 4

CyberArk notification system creates email.

```
CyberArk Notification
        |
        |
        v
SMTP Server
```

---

## Step 5

Admin receives:

```
From:
CyberArk

Subject:
Password Change Failed

Account:
Administrator

Reason:
Invalid credentials
```

# 7. What triggers an email?

CyberArk uses **events**.

Think:

```
EVENT happens
        |
        v
CyberArk checks rules
        |
        v
Should I send email?
        |
        v
Send notification
```

Examples:

---

## Event:

User requests account access

```
User
 |
 v
PVWA
 |
 v
Approval workflow
 |
 v
Email manager
```

Email:

```
John requested access to Root account.
Please approve.
```

---

## Event:

Password expires soon

```
Account password expires in 5 days
```

CyberArk:

```
Generate notification
```

Email:

```
Password expiration warning
```

---

## Event:

CPM cannot change password

```
CPM
 |
 |
Password change failed
```

Email:

```
CPM Error Alert
```

# 8. Does CyberArk store emails?

No.

CyberArk does **not** store emails.

The flow is:

```
CyberArk
   |
   |
Creates email message
   |
   |
SMTP Server
   |
   |
Email system stores it
```

The mailbox belongs to:

- Microsoft Exchange
    
- Outlook
    
- Gmail
    
- Office 365
    
- etc.

---

The main thing to remember:

> **CyberArk is not an email server. CyberArk creates notifications and sends them to an SMTP server, and the SMTP server delivers the email.**

This is the same idea as CPM, PSM, and Vault communication: each CyberArk component has a specific job, and they communicate through defined services.
