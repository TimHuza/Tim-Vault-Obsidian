Imagine you have a **big school** with thousands of students. If everyone was in one giant list, it would be very hard to manage. So the school creates groups:

- Grade 6 students
    
- Grade 7 students
    
- Teachers
    
- Administration
    
- Sports team
    

This makes it easier to organize and control things.

In **Active Directory (AD)**, the same idea is called an **Organizational Unit (OU)**.

## What is an Organizational Unit (OU)?

An **Organizational Unit (OU)** is a **container (folder) inside Active Directory that is used to organize users, computers, groups, and other objects**.

Think of an OU as a **folder in Windows Explorer**.

Example:

```
Company
│
├── Users
│   ├── John
│   ├── Sarah
│
├── Servers
│   ├── Server01
│   ├── Server02
│
└── Workstations
    ├── PC001
    ├── PC002
```

Here:

- `Users` is an OU
    
- `Servers` is an OU
    
- `Workstations` is an OU
    

Each OU contains objects.

---

# Why do we need OUs?

Without OUs, Active Directory would look like this:

```
Active Directory
│
├── John
├── Server01
├── PC001
├── Sarah
├── Server02
├── PC002
├── AdminAccount
```

Very messy!

With OUs:

```
Active Directory
│
├── Employees OU
│     ├── John
│     └── Sarah
│
├── Servers OU
│     ├── Server01
│     └── Server02
│
└── Computers OU
      ├── PC001
      └── PC002
```

Now administrators can easily manage things.

---

# How are OUs used with CyberArk?

CyberArk works closely with **Active Directory** because many privileged accounts are AD accounts.

For example:

A company might have:

```
Active Directory
│
├── Domain Admins OU
│       ├── Administrator
│       └── BackupAdmin
│
├── Server Admins OU
│       ├── ServerAdmin01
│       └── ServerAdmin02
│
└── Service Accounts OU
        ├── SQL_Service
        └── IIS_Service
```

CyberArk can use these OUs to find and manage accounts.

---

# Example: CyberArk onboarding an account

Imagine you have this AD account:

```
Username:
SQL_Service

Location:
Active Directory
      |
      └── Service Accounts OU
              |
              └── SQL_Service
```

CyberArk can:

1. Discover the account from Active Directory
    
2. Add it into a Safe
    
3. Store its password in the Vault
    
4. Rotate the password using CPM
    
5. Allow users to access it through PVWA/PSM

# OUs and Group Policies (GPO)

One important thing:

**OUs are also used for applying Group Policies.**

Example:

You have:

```
Servers OU
│
├── SQL Server 1
├── SQL Server 2
```

You can apply a GPO:

```
Servers OU
      |
      └── Password Policy:
             - Minimum password length: 20
             - Disable USB devices
             - Enable auditing
```

Every computer inside that OU receives the policy.

# OU vs Group (common beginner confusion)

They look similar but they are different.

## OU = Organization folder

Used for:

✅ Organizing objects  
✅ Applying GPO policies  
✅ Delegating administration

Example:

```
OU: Database Servers

Contains:
- SQL01
- SQL02
- SQL03
```

---

## Group = Permission collection

Used for:

✅ Giving access rights

Example:

```
Group:
Database Administrators

Members:
- John
- Sarah
- Mike
```

The group gives permissions.

## Simple definition to remember:

**An Organizational Unit (OU) is like a folder inside Active Directory that organizes users, computers, and accounts so administrators can easily manage them and apply policies.**

For CyberArk:

> **OUs help CyberArk find and manage privileged accounts that live inside Active Directory.**
