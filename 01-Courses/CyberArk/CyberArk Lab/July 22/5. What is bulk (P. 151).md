#cyberark-labs 


In CyberArk, **Bulk** means doing the same action on **many accounts at the same time** instead of doing it one account at a time.

Think about it like this:

Imagine you have **100 doors** in a building, and every door has a different key. If you need to change all the locks, you have two choices:

### Without Bulk

You change one lock → then the next → then the next...

```
Door 1  → Change key
Door 2  → Change key
Door 3  → Change key
...
Door 100 → Change key
```

This takes a long time.

### With Bulk

You say:

> "CyberArk, change the password for all these accounts."

CyberArk does the work for all selected accounts.

```
             CyberArk
                |
      ---------------------
      |        |          |
   Account1 Account2 Account3
      |        |          |
   Change   Change    Change
   password password password
```

---

# Where do we use Bulk in CyberArk?

Usually in **PVWA (Password Vault Web Access)**.

Examples:

- Add many accounts to a Safe
    
- Change properties of many accounts
    
- Delete many accounts
    
- Move many accounts
    
- Set permissions for many accounts
    
- Trigger password changes for many accounts
    

---

# Example from a CyberArk lab

Imagine you have these accounts:

```
Safe: Windows_Admin

Accounts:

Administrator@Server01
Administrator@Server02
Administrator@Server03
Administrator@Server04
```

You want CyberArk to manage all of them.

Without Bulk:

You open every account:

```
Account 1 → Edit → Change settings
Account 2 → Edit → Change settings
Account 3 → Edit → Change settings
Account 4 → Edit → Change settings
```

Very slow.

With Bulk:

You select:

```
☑ Administrator@Server01
☑ Administrator@Server02
☑ Administrator@Server03
☑ Administrator@Server04
```

Then:

```
Actions → Bulk Update
```

CyberArk applies the change to all accounts.

# How does Bulk work internally?

The flow looks like this:

```
Administrator
      |
      |
      v
PVWA
(you select many accounts)
      |
      |
      v
CyberArk Vault
(checks permissions)
      |
      |
      v
CPM
(Central Policy Manager)
      |
      |
      v
Target Systems
(Windows/Linux/Database/etc.)
```

---

## Step 1: You select accounts

Example:

```
Account A
Account B
Account C
```

PVWA creates a bulk request.

---

## Step 2: CyberArk checks permissions

CyberArk asks:

> "Is this user allowed to change these accounts?"

It checks:

- Safe permissions
    
- Account permissions
    
- Your CyberArk role
    

For example:

```
User Tim

Safe Permission:
✔ Use Accounts
✔ Manage Accounts
✔ Update Account Content
```

Allowed.

---

## Step 3: CyberArk applies the action

Example: Change password.

The request goes to:

```
CPM
```

The CPM then talks to the target systems.

Example:

```
CPM
 |
 |
 v
Windows Server
 |
 |
Change Administrator password
```

---

# Bulk vs Automatic Password Management (APM)

These are different.

## Bulk

Means:

> "Do this action on many accounts now."

Example:

```
Change passwords for 500 accounts immediately
```

---

## Automatic Password Management

Means:

> "CyberArk automatically manages passwords according to policy."

Example:

```
Every 30 days:
        |
        v
      CPM
        |
        v
Change password automatically
```

---

# Bulk Password Change Example

Imagine your company has:

```
500 Domain Admin accounts
```

Security policy says:

> "All privileged passwords must change today."

Instead of waiting for CPM schedules:

Administrator:

```
Select 500 accounts

        ↓

Bulk Password Change

        ↓

CPM changes all passwords
```

---

# Important things to remember

|Term|Meaning|
|---|---|
|Bulk|Perform an action on many accounts together|
|PVWA|Where the administrator starts the bulk operation|
|Vault|Stores account information securely|
|CPM|Executes password management actions|
|Safe permissions|Decide if you are allowed to do bulk actions|

---

A simple way to remember:

> **Bulk = "Do the same thing to many CyberArk objects at once."**

It is like selecting many files on your computer and clicking **Delete** once instead of deleting each file separately.
