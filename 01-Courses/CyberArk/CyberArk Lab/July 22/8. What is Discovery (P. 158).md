Imagine you have a **big school building** with hundreds of lockers. Some lockers contain important things, but nobody knows:

- Which lockers exist
    
- Who owns them
    
- Which ones have valuable items
    
- Which keys are shared
    

Before you can protect the lockers, you need someone to **walk around and make a list**.

In CyberArk, **Discovery** is that person walking around and making the list.

# What is Discovery in CyberArk?

**Discovery is a CyberArk feature that finds privileged accounts, machines, and credentials in your environment that are not yet managed by CyberArk.**

In simple words:

> **Discovery helps CyberArk find accounts that should be protected.**

Many companies have thousands of accounts:

- Windows administrator accounts
    
- Domain administrator accounts
    
- Service accounts
    
- Database admin accounts
    
- Linux root accounts
    
- Application accounts
    

Some of these accounts may have powerful permissions but nobody knows about them.

Discovery finds them.

---

# Why do we need Discovery?

Before CyberArk can protect an account, the account must be:

1. Found
    
2. Added into CyberArk
    
3. Stored in a Safe
    
4. Managed by CPM
    
5. Accessed through PVWA/PSM
    

The problem is:

> How do we protect accounts if we don't know they exist?

Discovery solves this problem.

---

# How does CyberArk Discovery work?

Let's look at the pieces.

## 1. Discovery Scanner

CyberArk uses a component called:

**CyberArk Discovery Scanner**

It performs scans against your environment.

It can search:

- Active Directory
    
- Windows machines
    
- Linux/Unix machines
    
- Databases
    
- Network devices
    

---

## 2. Scanner connects to targets

Example:

Your company has:

```
CyberArk Server

      |
      |
      v

Windows Server 1
Windows Server 2
Domain Controller
SQL Server
```

The scanner connects and checks:

- What computers exist?
    
- What users exist?
    
- Which accounts have admin privileges?
    
- Which passwords are weak or old?
    

---

## 3. Discovery collects information

It gathers information like:

### Account name

Example:

```
Administrator
```

### Account type

Example:

```
Windows Domain Account
```

### Location

Example:

```
DOMAIN\Administrator
```

### Privileges

Example:

```
Member of:
Domain Admins
```

### Password information

Depending on configuration:

- Password age
    
- Last change date
    
- Password policy status
    

---

## 4. Results appear in CyberArk

The administrator can view discovered accounts.

Example:

```
Discovery Results

Account             Risk

Administrator       High
SQL_Admin           High
Backup_Service      Medium
Test_User           Low
```

Then they decide:

```
Onboard Account
        |
        v
Create CyberArk Account Object
        |
        v
Store in Safe
        |
        v
Start managing password
```

# Discovery vs Automatic Account Onboarding

These two are often confused.

## Discovery

Finding accounts.

Example:

"CyberArk, tell me what accounts exist."

Output:

```
Found:
Administrator
SQL_Admin
Backup_Service
```

---

## Automatic Onboarding

Adding those accounts into CyberArk automatically.

Example:

```
Discovery finds account

        ↓

Rules check account

        ↓

Account added to Safe
```

# What is the purpose of Discovery?

The main purposes are:

### 1. Find unknown privileged accounts

"Show me every powerful account."

---

### 2. Reduce security risks

Find accounts like:

```
Administrator
Password never changed
Domain Admin
```

---

### 3. Prepare accounts for onboarding

Discovery creates a list of accounts ready to put into CyberArk.

---

### 4. Help with audits

Security teams can answer:

"Do we know all privileged accounts in our company?"

---

# Discovery vs CPM vs PSM

Beginners often mix these up:

|Component|Job|
|---|---|
|**Discovery**|Finds accounts|
|**CPM**|Changes and manages passwords|
|**PVWA**|Web interface where users manage accounts|
|**PSM**|Controls and records sessions|
|**Vault**|Stores passwords securely|

Think of it like:

```
Discovery = Finds the treasure

Vault = Stores the treasure

CPM = Changes the lock combination

PSM = Watches who opens the treasure box
```

---

## Short definition to remember:

> **CyberArk Discovery scans your environment to find privileged accounts and assets that are not yet managed by CyberArk, so they can be reviewed and onboarded for protection.**

In a real CyberArk deployment, Discovery is usually one of the first steps because you cannot protect privileged accounts if you don't know where they are.
