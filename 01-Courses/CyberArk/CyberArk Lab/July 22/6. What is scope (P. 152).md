#cyberark-labs 


In CyberArk, **Scope** means:

> **"Which objects, accounts, users, or systems does this rule, permission, or setting apply to?"**

Think of **scope as a boundary or a target area**.

Imagine you have a school:

- The **whole school** = everything that exists
    
- The **math classroom** = a smaller area
    
- **One student** = an even smaller area
    

The question is:

**"Where should this rule work?"**

That is the **scope**.

---

## Simple CyberArk example

Imagine you have a Safe called:

```
Windows Admin Accounts
```

Inside the Safe you have:

```
Server01\Admin
Server02\Admin
Server03\Admin
```

Now you create a user:

```
John
```

and give him permission.

You need to decide:

### Scope 1: All accounts

John can access:

```
Server01\Admin
Server02\Admin
Server03\Admin
```

The scope is:

```
All accounts in this Safe
```

---

### Scope 2: Only one account

John can access only:

```
Server01\Admin
```

The scope is:

```
Only Server01\Admin
```

---

# Scope in CyberArk Permissions

A lot of CyberArk features use scope.

For example:

## 1. Safe permissions

When you add a user to a Safe:

```
Safe: Production Servers

User: Tom

Permissions:
☑ Retrieve accounts
☑ Use accounts
☑ List accounts
```

The scope is:

```
Production Servers Safe
```

Meaning:

Tom's permissions only apply inside that Safe.

---

## 2. Platforms

Remember:

A **Platform** defines rules for accounts.

Example:

```
Windows Domain Account
```

has settings:

```
Change password every 30 days
Verify password every 7 days
Use CPM plugin
```

The scope of this Platform is:

```
All accounts assigned to Windows Domain Account platform
```

Example:

```
Account A  ---> Windows Domain Account Platform
Account B  ---> Windows Domain Account Platform
Account C  ---> Windows Domain Account Platform
```

All three receive the same rules.

---

## 3. CPM Password Management

Example:

You configure:

```
Change password every 7 days
```

The scope could be:

```
Only Domain Admin accounts
```

or:

```
All Windows accounts
```

Depending on where the rule is applied.

---

## 4. PSM Access

Example:

You create a PSM connection rule:

```
Allow SSH connections
```

Scope could be:

```
Only Linux servers
```

Not:

```
All servers
```

---

# Scope vs Permission (important difference)

Beginners often confuse these.

### Permission = What can you do?

Example:

```
Retrieve password
Use account
Manage Safe
```

### Scope = Where does it apply?

Example:

```
Only this Safe
Only this account
Only this platform
Only this server
```

---

Think of it like a school:

|Concept|Example|
|---|---|
|Permission|"Can open the classroom door"|
|Scope|"Only the science classroom"|

Permission says **what action**.

Scope says **where that action works**.

---

# CyberArk example from a lab

You might see:

```
User: CyberArk_Admin

Permission:
Manage Safe

Scope:
Safe01
```

Meaning:

CyberArk_Admin can manage:

```
Safe01 ✅
```

but not:

```
Safe02 ❌
Safe03 ❌
```

because the scope is limited.

---

## Easy definition to remember

**Scope = The area where something applies.**

In CyberArk:

- Safe scope → which Safe?
    
- Account scope → which account?
    
- Platform scope → which accounts use these rules?
    
- User scope → which users receive permissions?
    

Whenever you see **Scope** in CyberArk, ask:

> "What things will this setting affect?" ✅
