Think of **Automated Release by PSM** as a **safety feature that automatically ends a privileged account session when the user closes the PSM connection or leaves without properly logging out**.

First, let's understand the problem it solves.

---

## 1. The problem: Someone checks out an account

Imagine CyberArk stores a powerful administrator account:

```
Username: Administrator
Password: ********
```

A user wants to use it.

Normally:

1. User requests access in PVWA.
    
2. CyberArk gives them access.
    
3. User connects through PSM.
    
4. They work on the server.
    

Example:

```
User
 |
 | Request account
 ↓
PVWA
 |
 | Gives access
 ↓
PSM
 |
 | Connects
 ↓
Windows Server
```

While the user is connected:

```
Administrator account = Locked for this user
```

Nobody else can use it.

---

## 2. The problem if the user disappears

Imagine:

- User opens a PSM session.
    
- They are using the Administrator account.
    
- Their computer crashes.
    
- They close the browser.
    
- They forget to log out.
    

CyberArk thinks:

> "The account is still being used."

So the account remains checked out.

This creates a problem:

```
Administrator account
        |
        ↓
Still assigned to User A
        |
        ↓
User B cannot use it
```

---

# 3. What is "Automatic Release by PSM"?

**Automatic Release by PSM** means:

> "If PSM detects that the user's session has ended, automatically check the account back into CyberArk."

In simple words:

PSM says:

> "The user is no longer connected. I will release the account."

---

Example:

### Without Automatic Release:

```
10:00 AM
User checks out Administrator account

10:30 AM
User closes browser accidentally

11:00 AM
Account still locked ❌
```

An administrator must manually click:

```
Check-in
```

---

### With Automatic Release:

```
10:00 AM
User checks out Administrator account

10:30 AM
User disconnects

PSM detects session ended

10:31 AM
CyberArk automatically releases account ✅
```

---

# 4. How does it work internally?

The process looks like this:

```
                 CyberArk Vault
                      |
                      |
                 Account locked
                      |
                      ↓
User ---> PVWA ---> PSM ---> Target Server
                    |
                    |
              PSM monitors session
                    |
                    ↓
          User disconnects
                    |
                    ↓
          PSM sends release request
                    |
                    ↓
             CPM checks account in
```

Important components:

### PVWA

The website where users request accounts.

### PSM

The machine that creates and monitors the privileged session.

### Vault

Stores the account and controls ownership.

### CPM

Changes passwords and manages account lifecycle.

# 6. Important parameters

The exact names can differ depending on CyberArk version, but common parameters are:

---

## Parameter 1: Allow automatic release by PSM

Example:

```
PSMReleasesAccount = Yes
```

Meaning:

```
PSM can automatically check-in the account
```

If disabled:

```
PSM session ends
       |
       ↓
Account stays checked out
```

---

## Parameter 2: Connection Component

Example:

```
PSM-WinServer
PSM-SSH
PSM-RDP
```

This tells CyberArk:

> "Which PSM connection type should control the release?"

Example:

For Windows:

```
PSM-RDP
```

For Linux:

```
PSM-SSH
```

---

## Parameter 3: Session timeout

Example:

```
Max Session Duration
=
2 hours
```

Meaning:

User can use the account for maximum 2 hours.

After:

```
2 hours finished
        |
        ↓
PSM terminates session
        |
        ↓
Account released
```

---

## Parameter 4: Idle timeout

Example:

```
Idle Timeout = 15 minutes
```

Meaning:

If the user does nothing:

```
No keyboard
No mouse
No activity

15 minutes passes

PSM disconnects user
```

Then:

```
Account released
```

---

# 7. Difference between Automatic Release and Check-In

This is important.

## Check-In button

Manual:

```
Administrator clicks:

Account
 |
 Check-In
```

Someone must do it.

---

## Automatic Release by PSM

Automatic:

```
PSM detects session ended

      ↓

CyberArk releases account
```

Nobody needs to click anything.

# 9. Why companies use it?

Because privileged accounts are dangerous.

Example:

```
Domain Administrator
Root account
Database administrator
Cloud admin
```

If someone keeps one checked out:

- Other admins cannot work
    
- Security risk increases
    
- Password rotation may be blocked
    

Automatic release makes sure:

✅ Accounts do not stay locked  
✅ Less manual administration  
✅ Better security  
✅ Faster reuse of privileged accounts

---

## Simple one-sentence definition:

**Automatic Release by PSM is a CyberArk feature where PSM automatically checks a privileged account back into the Vault when the user's privileged session ends, preventing accounts from staying locked accidentally.**
